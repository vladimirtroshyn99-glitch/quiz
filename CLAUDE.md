# Квиз-лид-магнит — CLAUDE.md

## Описание проекта
Мобильное веб-приложение-квиз. Lead magnet для эксперта-родолога. Аудитория — женщины.

## Стек
- **Next.js 14** — Pages Router (НЕ App Router)
- **Tailwind CSS v3** — стилизация
- **Render** — деплой

## Структура
- `pages/index.js` — **весь** фронтенд и стейт приложения в одном файле
- `pages/_app.js` — только подключение глобальных стилей
- `styles/globals.css` — Tailwind base + глобальные сбросы

## Правила кода
- Не плодить компоненты без жёсткой необходимости
- Простой, чистый код без оверинжиниринга
- Никаких TypeScript, тестов, лишних зависимостей

## Текущий статус
**Этап 2 завершён: Экран сбора запроса**

### Что реализовано в Этапе 1:
- Стартовый экран: логотип-заглушка, заголовок-приветствие, 4 карточки тем
- Клик по карточке → сразу переход на Экран 2 (без лишней кнопки «Далее»)
- **Instagram/Facebook WebView detection + auto-escape:**
  - Android → автоматический редирект в Chrome через `intent://`
  - iOS → попытка `x-safari://` deeplink, при неудаче — full-screen overlay с инструкцией
  - Остальные случаи → тонкий баннер вверху

### Что реализовано в Этапе 2:
- Управление экранами через `screen` state: `'start'` | `'input'` | `'result'`
- Экран 2: кнопка «Назад», индикатор выбранной сферы (пилюля с эмодзи), textarea, голос
- Голосовой ввод: Web Speech API, `ru-RU`, continuous — диктовка дописывается в textarea
- Логика блокировки кнопки «Продолжить»: disabled если поле пустое ИЛИ идёт запись
- Кнопка остановки записи — нейтральный серый тон (не красный)
- Экран 3 — заглушка результата с кнопкой «В начало»
- Данные (сфера + текст) логируются в `console.log` для будущей интеграции с ИИ

## История этапов
- **Этап 1** — каркас, стартовый экран, WebView escape
- **Этап 2** — экран ввода запроса, голосовой ввод, навигация между экранами

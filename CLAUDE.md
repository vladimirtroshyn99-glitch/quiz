# Квиз-лид-магнит — CLAUDE.md

## Описание проекта
Мобильное веб-приложение-квиз. Lead magnet для эксперта-родолога. Аудитория — женщины.

## Стек
- **Next.js 14** — Pages Router (НЕ App Router)
- **Tailwind CSS v3** — стилизация
- **Render** — деплой

## Структура
- `pages/index.js` — **весь** фронтенд и стейт приложения в одном файле
- `pages/_app.js` — только подключение глобальных стилей
- `styles/globals.css` — Tailwind base + глобальные сбросы

## Правила кода
- Не плодить компоненты без жёсткой необходимости
- Простой, чистый код без оверинжиниринга
- Никаких TypeScript, тестов, лишних зависимостей

## Текущий статус
**MVP 0.5: методологический квиз по 5 фазам, 5-блочный разбор с визуализациями «Разрыв» и «Корень», попап-чат, лид-форма → Vakas-вебхук, финальная развилка «Да/Нет»**

**Корректировка User Flow:** Блок 5 (block5_dig, «Куда копать») перенесён в секцию после заполнения лид-формы как переход к финальному офферу. На основном экране результатов отображаются только Блоки 1–4 + диаграммы. После отправки лид-формы: CTA-кнопки скрываются, Блок 5 плавно раскрывается (анимация max-height + opacity), под ним — финальный вопрос «Записаться?» с кнопками «Да, хочу» (vakas + экран ассистента) и «Нет, удалите мои ответы» (экран telegram). Поле `action_step` из JSON больше не отображается на фронтенде.

### Структура квиза (MVP 0.5):
- Фиксированные варианты ответов для всех 10 вопросов (QUIZ_OPTIONS): «Прям в точку» / «Отчасти» / «Раньше было, сейчас меньше» / «Не совсем про меня»
- Фаза 1 «Зеркало» (1–3): возвращают точку А, человек чувствует «меня видят»
- Фаза 2 «Мечта» (4–5): желаемое состояние, эмоциональный разрыв
- Фаза 3 «Блок» (6–7): скрытые корневые затыки
- Фаза 3.5 «Цена» (8): утверждение-гипотеза о последствии бездействия (НЕ открытый вопрос — ИИ сам прогнозирует тяжёлое следствие; пользователь калибрует «Прям в точку» / «Отчасти» и т.д.)
- Фаза 4 «Готовность» (9–10): уровень готовности действовать

### JSON-схема разбора (result.js):
- **block1_see** — «Вот что я вижу» (3–4 предл., текущая ситуация, БЕЗ советов)
- **block2_want** — «Вот чего ты на самом деле хочешь» (3–4 предл., БЕЗ советов)
- **chart_gap** `{current: 10-45, desired: 80-97}` → визуализация «Разрыв»
- **block3_hold** — «Вот что тебя держит» (3–4 предл., БЕЗ советов)
- **chart_root** `{surface, deep, root}` → визуализация «Корень» (3-уровневый стек)
- **block4_lose** — «Вот что ты теряешь» (2–3 предл., цена бездействия)
- **block5_dig** — «Куда копать» (2–3 предл., направление в сторону рода)
- **suggested_questions** — 3 вопроса от первого лица (попап-чат)
- **cta_button_targeted** — персонализированный крючок-вопрос (до 10 слов)
- **action_step** — практические шаги (скрыто до отправки формы)

### Что реализовано в Этапе 1:
- Стартовый экран: логотип-заглушка, заголовок-приветствие, 4 карточки тем
- Клик по карточке → сразу переход на Экран 2 (без лишней кнопки «Далее»)
- **Instagram/Facebook WebView detection + auto-escape:**
  - Android → автоматический редирект в Chrome через `intent://`
  - iOS → попытка `x-safari://` deeplink, при неудаче — full-screen overlay с инструкцией
  - Остальные случаи → тонкий баннер вверху

### Что реализовано в Этапе 2:
- Управление экранами через `screen` state: `'start'` | `'input'` | `'result'`
- Экран 2: кнопка «Назад», индикатор выбранной сферы (пилюля с эмодзи), textarea, голос
- Голосовой ввод: Web Speech API, `ru-RU`, continuous — диктовка дописывается в textarea
- Логика блокировки кнопки «Продолжить»: disabled если поле пустое ИЛИ идёт запись
- Кнопка остановки записи — нейтральный серый тон (не красный)
- Экран 3 — заглушка результата с кнопкой «В начало»
- Данные (сфера + текст) логируются в `console.log` для будущей интеграции с ИИ

### Что реализовано в Этапе 3:
- `pages/api/analyze.js` — Claude Sonnet 4.6 с системным промптом эксперта-родолога
- Логика: короткий запрос → 1 уточняющий вопрос → тест; содержательный → сразу тест
- Экран загрузки с цикличными сообщениями и анимацией
- Экран уточнения: карточка с вопросом ИИ, textarea + голос, кнопка «Ответить»
- Экран превью теста: 10 персонализированных вопросов с 4 вариантами ответов
- Prompt caching на системном промпте (эфемерный кэш Anthropic)

### Что реализовано в Этапе 4:
- Пошаговый экран теста: один вопрос на экран, счётчик «Вопрос N из 10»
- Прогресс-бар (тонкая полоска, анимированная через transition)
- Авто-переход на следующий вопрос через 380 мс после тапа
- Кнопка «← Назад» для возврата к предыдущему вопросу (убирает последний ответ из массива)
- Защита от двойного тапа во время задержки перехода
- Экран анализа после последнего вопроса: пульсирующие кольца, цикличные сообщения
- Кнопка «В начало» на экране анализа появляется через 10 сек (страховка от зависания)

### Что реализовано в Этапе 6:
- `pages/api/chat.js` — Claude Sonnet 4.6, системный промпт с полным контекстом разбора
- Приветственное сообщение генерируется автоматически при входе в чат (пустая history)
- Флаг `isLast: true` в 3-м запросе — Claude мягко подводит к живому специалисту
- Prompt caching на системном промпте
- Экран чата: аватар, лента облаков (серые ИИ / розовые пользователь), авто-скролл
- `questionCount` — счётчик сообщений пользователя (0–3)
- После 3-го вопроса: поле ввода блокируется «Лимит исчерпан», появляется кнопка «Узнать, что делать дальше» → `screen: 'offer'` (Этап 7)

## История этапов
- **Этап 1** — каркас, стартовый экран, WebView escape
- **Этап 2** — экран ввода запроса, голосовой ввод (Whisper), навигация между экранами
- **Этап 3** — Claude Sonnet, логика уточнения, генерация структуры теста
- **Этап 4** — интерактивный квиз, прогресс-бар, сбор ответов, экран ожидания результатов
- **Этап 5** — /api/result.js, анимированные Tailwind-графики, кнопка перехода в чат
- **Этап 6** — /api/chat.js, микро-чат с лимитом 3 вопроса, прогрев к целевому действию
- **Этап 7** — экран 'offer', 'assistant' (кодовое слово «РОД»), 'telegram' (канал)
- **MVP 0.5** — квиз по 5 фазам методологии, фиксированные ответы, новая структура разбора (5 блоков + 2 диаграммы), попап-чат, лид-форма, Vakas-вебхук, action_step
